Oracle备份与恢复

  1. 表、索引、视图等（数据层）
    表select from user_tables
    视图select     from user_views
    索引select from user_indexes;
    索引与表的关系 select     from user_ind_columns;
  2. 用户（逻辑层） select * from dba_users;
    通俗的讲就是访问oracle数据库的账号。
    在oracle中，可以对用户的各种安全参数进行控制，以维护数据库的安全性，这些概念包括模式(schema)、权限、角色、存储设置、空间限额、存取资源限制、数据库审计等。每个用户都有一个口令，使用正确的username/password才能登录到数据库进行数据存取。
  3. 表空间（逻辑层） select * from v$tablespace;
    ORACLE数据库被划分成称作为表空间的逻辑区域——形成ORACLE数据库的逻辑结构。一个ORACLE数据库能够有一个或多个表空间,而一个表空间则对应着一个或多个物理的数据库文件。表空间是ORACLE数据库恢复的最小单位,容纳着许多数据库实体,如表、视图、索引、聚簇、回退段和临时段等。

每个ORACLE数据库均有SYSTEM表空间,这是数据库创建时自动创建的。SYSTEM表空间必须总要保持联机,因为其包含着数据库运行所要求的基本信息(关于整个数据库的数据字典、联机求助机制、所有回退段、临时段和自举段、所有的用户数据库实体、其它ORACLE软件产品要求的表)。

  1. 数据文件（物理层） select * from DBA_DATA_FILES;
    每一个ORACLE数据库有一个或多个物理的数据文件(data file)。逻辑数据库结构(如表、索引)的数据物理地存储在数据库的数据文件中。
  2. 关系解释
    物理层>>逻辑层<<数据层
    数据一定有唯一的逻辑归属（表一定属于某个确定的用户和某个确定的表空间）
    用户和表空间同为逻辑层的概念，没有绝对的物理文件体现也没有绝对的从属关系（用户不属于某个表空间、表空间也不属于某个用户）
    物理文件一定有唯一的逻辑归属（数据文件一定属于某个确定的表空间，反之不成立；数据文件与用户没有直接关联）
    在创建表、索引等数据时需要指定用户和表空间，当没有指定用户时默认为当前登录的用户，当没有指定表空间时默认为用户的默认表空间（习惯上将用户与表空间的名称相同便于管理，存在用户和表空间不同的情况，分表空间存放同一用户的不同数据一般是为了解决物理存储空间不够问题和提高查询速度）。
    查询表、视图、索引等时需要指定用户不需要指定表空间，表空间在创建的时候已经指定过了，当没有指定用户时默认为当前登录的用户（用户查询其它用户的表需要相应的权限，对于通过角色获取的权限可以查询其它用户的数据，但在调用其它用户的内部过程中不会继承该用户的角色权限，只能继承对应用户直接拥有的权限）。
    存在以下情况:
    一个表空间拥有多个数据文件；
    一个用户拥有多个表或其它数据；
    一个用户的数据存放在多个表空间；
    一个表空间存放多个用户数据；
    不存在以下情况：
    两个表空间共用同一个数据文件；
    两个用户同时拥有一份数据；

备份与还原

一、ORACLE DIRECTORY目录
Create DIRECTORY让我们可以在Oracle数据库中灵活的对文件进行读写操作，极大的提高了Oracle的易用性和可扩展性。
创建 CREATE [OR REPLACE] DIRECTORY directory AS ‘pathname’;
授权 GRANT READ[,WRITE] ON DIRECTORY directory TO username;
查询 select * from DBA_ DIRECTORIES;
简单的理解为将系统的某个文件夹作为数据库用户可以操作的位置，并给每个可操作的目录一个别名，DIRECTORY目录是存放在sys下的，可以给不同用户赋予对目录不同的操作权限。
用途：备份还原的文件存放、LOB文件、控制文件等文件的导出导入时存放。
注意：这里创建的目录实际的位置是所连接的Oracle实例的服务端所在的服务器上的路径，服务端是Linux的可以在Windows客户端进行目录的创建和修改，但是要按照服务端操作系统的格式来填写相应的值。

二、备份
习惯上说备份某个库是导出数据库某个用户下的全部数据的意思。
备份常用的有三种方式exp程序、expdp程序和PLsql提供的导出表工具。
exp程序和expdp程序就是数据泵程序，是oracle官方提供的专门用于oracle数据库不加密导出的工具。其中exp程序是早期版本可以附加参数较少，一般使用新的expdp工具，两种工具的导出结果不能通用需要注意。（只总结expdp的相关参数，exp的较为简单不赘述）

  1. PLsql 提供的导出表工具使用方法：
    打开PLsql登录,位置在 "工具">"导出表…" ，在该工具中选择需要导出的表，选择希望导出后的存储格式对应的选项卡，设置相应的参数，输出文件的路径要选择其它参数可以使用默认的。
    注意：这里的导出是远程导出、只能导出当前登录用户的表、不能完全备份用户的全部数据。
    远程导出会把数据从服务端的数据文件抽取到服务端交互空间，再通过网络传输到客户端交互空间，最后再转存到指定的客户端的输出文件中，过程中不使用DIRECTORY目录，但因为通过网络服务传输速度会慢很多尤其在数据量较大时。

  2. 三、 还原

  3. 正常状态下的迁移、复制、还原
    还原过程与导出过程相同，这里不在赘述相同部分。
    备份与导出的意思是将oracle用户的数据部分或者全部从数据库服务中拷贝到一个与数据库服务没有关系的文件中，同时该文件是非加密的开放的数据一般将文件后缀定义为dmp或dump并没有绝对限制，一般分为通过网络的远程导出（PLsql提供的工具、异地备份方案和OGG）和通过服务的任务型备份（expdp、exp），两种方式都要求服务端运行正常。
    迁移、复制和还原的意思是将数据从一个oracle服务中导入到另一个oracle服务中，要求后者拥有的空间不小于前者需要转存的部分。后者和前者可以是同一个oracle服务。
    一般情况下为了简化转存的过程要求两个服务涉及到需要转存部分的逻辑层结构相同，当然也可以不同，对于不同的情况需要有部分需要注意的地方在后文中会强调。

  5. 2）IMPDP导入工具
    与EXPDP导出程序一样是在命令提示符窗口执行命令即可，基本上EXPDP的参数在IMPDP中都适用，但IMPDP还包含很多额外的参数总结如下：

| 关键字 | 说明 (默认)
| REMAP_DATAFILE | 在所有 DDL 语句中重新定义数据文件引用。
| REMAP_SCHEMA | 将一个方案中的对象加载到另一个方案。
| REMAP_TABLESPACE | 将表空间对象重新映射到另一个表空间。
| REUSE_DATAFILES | 如果表空间已存在, 则将其初始化 (N)。
| SKIP_UNUSABLE_INDEXES | 跳过设置为无用索引状态的索引。
| SQLFILE | 将所有的 SQL DDL 写入指定的文件。
| STREAMS_CONFIGURATION | 启用流元数据的加载
| TABLE_EXISTS_ACTION | 导入对象已存在时执行的操作。有效关键字: (SKIP), APPEND, REPLACE 和 TRUNCATE。
| TRANSFORM | 要应用于适用对象的元数据转换。有效的转换关键字: SEGMENT_ATTRIBUTES, STORAGE OID 和 PCTSPACE。
| TRANSPORT_DATAFILES | 按可传输模式导入的数据文件的列表。

REMAP_SCHEMA 可以定义用户的切换，其格式为：

  • remap_schema=old_schema_name:new_schema_name
    REMAP_TABLESPACE 可以定义切换对象的不同表空间，其格式为：
  • remap_tablespace=old_tablespace_name:new_tablespace_name
    参数中还包含了屏蔽索引和键值，DLL错误的应对机制等功能（当需要创建的表已经存在的时候等），使用时与其它参数模式相同。这些特有的参数都可以放在参数文件中调用。

  1. 异常状态下的恢复工作
  • ORACLE冷备份及恢复
    备份
    直接拷贝oracle目录下的admin、oradata(datafile， controlfile，redo)、flash_recovery_area三个文件夹，db_1目录下database(PWDfile、pfile)、dbs(spfile)、NETWORK/ADMIN(listener.ora、tnsnames.ora)，到其他存储实现备份。
    恢复
    方法一：最简单的方法（需建库）
    对于清楚原库安装过程的方法一较为简单
    (1).创建一个和原来一样的数据库。(安装路径和数据库名必须和原来一致)
    (2).停止数据库 shutdown immediate；
    (3). 复制安装目录下的admin、oradata、flash_recovery_area覆盖，复制database(PWDfile、pfile) 覆盖
    (4).启动数据库 startup;
    方法二：（不需建库，稍麻烦点）
    (1).操作系统重装，如果做冷备恢复，要保证相同操作系统，相同的数据库版本。
    (2).正常安装oracle软件，只需要安装软件，不用建实例。
    (3).数据覆盖，包括数据文件、参数文件、控制文件、日志文件、pwd文件，放在与原系统相同的目录。如果目录有所改变，则需要另外建立控制文件，修改pfile。
    （对于需要通过冷备份恢复的数据库一般原数据库服务端出现不可修复错误时才这样，启动文件一般不容易找，可以通过对文件的查找自己修改PFILE文件，后文中有介绍PFILE内容）
    (4).建立服务：使用oradim 命令 cmd下 oradim -new -sid gbicc ，表示建立一个服务，sid为gbicc。如果是在linux下，不需要此步。
    (5).建立监听： net configuration assintant 来建立(建议将源系统的network下的文件拷过来，根据实际情况修改)。
    (6).打开数据库： cmd
    set oracle_sid=gbicc;
    sqlplus /nolog;
    Conn / as sysdba;
    startup;
    至此，冷备份恢复成功。原理简单实际操作中较为复杂。

  • SPFILE与PFILE 概念
    ORACLE中的参数文件是一个包含一系列参数以及参数对应值的操作系统文件，可以分为两种类型。它们是在数据库实例启动时候加载的，决定了数据库的物理结构、内存、数据库的限制及系统大量的默认值、数据库的各种物理属性、指定数据库控制文件名和路径等信息，是进行数据库设计和性能调优的重要文件。
    初始化参数文件（Initialization Parameters Files），Oracle 9i之前，ORACLE一直采用PFILE方式存储初始化参数，该文件为文本文件。
    服务器参数文件（Server Parameter Files），从Oracle 9i开始，Oracle引入了SPFILE文件，该文件为二进制格式，不能通过手工修改。
    PFILE是文本文件的，而SPFILE是二进制格式的。PFILE文件可以用文本编辑器打开手工配置、而SPFILE不行，只能通过SQL命令在线修改。从操作系统上可以看到这两者的区别，初始化参数文件为ASCII文本文件，SPFILE为数据文件。
    重建PFILE或SPFILE
    create spfile[=’xxxxx’] from pfile[=’xxxx’];
    create pfile[=’xxxxx’] from spfile[=’xxxx’];
    查询SPFILE的位置
    SELECT NAME, VALUE, DISPLAY_VALUE FROM V$PARAMETER WHERE NAME =’spfile’;
    如果VALUE值为非空，则是SPFILE启动，否则是PFILE。

    双机热备机器安装过程

  1. 分别按照主机A和备机B的oracle软件到各自的独立硬盘中；
  2. 将公共资源（共享硬盘、IP地址）分配给A；
  3. 在A上使用Database Configuration Assistant创建实例

  4. 创建过程中步骤6：数据库文件所在位置，选择所有数据文件使用公共位置。在共享硬盘中创建数据文件存放位置。

  5. 创建过程中步骤7：恢复配置，快速恢复区指定到共享硬盘中指定的快速恢复位置。

  6. 完成A上的实例创建，测试连接没有问题后关闭A的实例服务和监听程序。（出现监听问题单独重启配置监听即可）

  7. 将公共资源（共享硬盘、IP地址）分配给B；
  8. 删除创建A上实例在公共硬盘生成的数据文件和快速恢复文件（建议重命名父文件夹即可）。
  9. 在B上按照A一样的过程创建一个一样的实例并测试连接。
  10. 关闭B服务和监听，分配公共资源回到A，启动实例服务和监听测试。正常即完成，不正常通过克隆SPFILE文件复制B的参数到A。

克隆SPFILE文件（使用sys用户以sysdba登录）：

  1. 双机中调整到服务可运行的机器上，使用下面语句创建中间PFILE文件。
    服务使用SPFILE运行的：
    CREATE PFILE=’C:\INIT.ORA’ FROM SPFILE;
    服务使用PFILE运行的：
    CREATE PFILE=’C:\INIT.ORA’ FROM PFILE;
  2. 拷贝中间PFILE INIT.ORA到另一台机器上
  3. 双机中调整到服务不能正常运行的机器上，进入CMD中使用中间PFILE重启实例。
    connect / as sysdba
    shutdown immediate
    startup pfile=’C:\INIT.ORA’

  4. 测试连接通畅，使用下面语句通过PFILE文件重建实例启动文件。
    服务使用SPFILE运行的：
    CREATE SPFILE FROM PFILE=’C:\INIT.ORA’;
    服务使用PFILE运行的：
    CREATE PFILE FROM PFILE=’C:\INIT.ORA’;

  5. 四、 批处理的使用
    批处理(Batch)，也称为批处理脚本。顾名思义，批处理就是对某对象进行批量的处理，通常被认为是一种简化的脚本语言，它应用于DOS和Windows系统中。批处理文件的扩展名为bat 。
    通过批处理文件添加的任务也会因为命令提示符窗口的关闭而放弃后续没有执行的命令（复制进入命令提示符的最后一行如果没有复制换行符是不会执行最后一行的，批处理由于没有pause的情况下不会停留在命令提示符界面因此不存在这个问题）。
    数据库自动备份的批处理
    数据库备份.bat
    set bfrq=%date:~0,4%%date:~5,2%%date:~8,2%
    expdp BDCK/salis@orcl schemas=BDCK directory=dmp_dir1 dumpfile=BDCK%bfrq%.DMP logfile=BDCK%bfrq%.log version=11.2.0.1.0
    此处仅列出了一个用户的导出语句，可以在同一个批处理添加所有需要导出的用户。
