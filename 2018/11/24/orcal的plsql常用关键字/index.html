orcal的plsql常用关键字

2018-11-24
orcal

orcal的plsql操作：

1.整体的框架之查询操作

select 列名，带别名的列名，列值表达式或子查询
查询结果列名称（多个时用逗号隔开）
必要子句
from锻炼、表名或子查询
查询对象名称（多个时用逗号隔开、多个时带别名）
dual用于一个没有查询对象的色了传统语句
非必要子句
with别名 as 子查询
构造查询用的历史表（写在select前）
inner join 表名或子查询
内连接（左右表或子查询都存在时返回结果）
left join 表名或子查询
左连接（左表和子查询存在时就返回结果）
right join 表名或子查询
右连接（右表或子查询存在时就返回结果）
where 一个布尔运算表达式
筛选掉建的的布尔运算（单行数据）列值限制
group by 列名或带别名的列名
合并列分组聚合（多个时用逗号隔开）
having 一个布尔运算表达式
筛选条件的布尔运算（聚合函数）行值限制
order by/order by desc 一个可以作为结果列的列值表达式
正序排序/倒序排序（对字符串和数字类型的排序机制是不同的）desc在尾部
union/union all 与当前select返回的=结果列数量类型一致的select语句
排除重复拼接/完全拼接（结果列必须完全相同）

2.一些概念总体解释：

  • 数据查询语言DQL(Data QueryLanguage):
    用于检索数据库中的数据，主要是select语句
  • 数据操纵语言DML(data manipulation language)：
    它们是UPDATE、INSERT、DELETE，就象它的名字一样，这3条命令是用来对数据库里的数据进行操作的语言
  • 事务控制语言TCL(Transaction Control Language):
    用于维护数据的一致性，包括commit、rollback、savepoint三条语句。commit用于提交对数据库的更改，rollback语句用于取消对数据库的更改，savepoint则用于设置保存点。
  • 数据定义语言DDL(data definition language):
    DDL比DML要多，主要的命令有CREATE、ALTER、DROP等，DDL主要是用在定义或改变表(TABLE)的结构，数据类型，表之间的链接和约束等初始化工作上，他们大多在建立表时使用
  • 数据控制语言DCL(Data Control Language)：
    是数据库控制功能。是用来设置或更改数据库用户或角色权限的语句，包括(grant,deny,revoke等)语句。在默认状态下，只有sysadmin,dbcreator,db_owner或db_securityadmin等人员才有权力执行DCL
    3.一些操作
    1.daul做为空表的操作
    eg: 将sql操作当计算机用：
    select 10020 from dual;
    查询当前日期：
    select to_char(sysdate,’yyyy-mm-dd,hh24:mi:ss’) from dual;
    2.连接字符串
    eg:select 列1 | |列2 from 表1;
    select concat(列1,列2) from 表1;
    3.视图中不能使用order by，但可用group by取代来达到排序目的
    create view a as select b1,b2 from b group by b1,b2;
    4.删除重复行
    SELECT语句中使用ALL或DISTINCT选项来显示表中符合条件的所有行或删除其中重复的数据行，默认为ALL。使用DISTINCT选项时，对于所有重复的数据行在SELECT返回的结果集合中只保留一行。
    select distinct(列1) from 表1；
    5.限制返回的行数（sqlserver）
    使用TOP n [PERCENT]选项限制返回的数据行数，TOP n说明返回n行，而TOP n PERCENT时，说明n是表示一百分数，指定返回的行数等于总行数的百分之几。
    例如：
    代码:SELECT TOP 2     FROM testtable
    代码:SELECT TOP 20 PERCENT FROM testtable
    对于orcal是使用rownum,且只有=1或者小于任意值：
    SELECT     FROM testtable where 列1=“” and rownum<10

6.对于decode作为复合函数可以用于组合连接字段，
eg：decode（列，判断值，返回值1，返回值2）
当判断值不存在时返回返回值2，该复合函数还可以继续加判断值和返回值。

8.分组查询
（1）select a,b from c group by a,b order by a;
查询c表，对a,b进行分组，列中的元素根据a值进行从小到大排列，同一类型的b分组在一起。
（2）select a,avg(b),sum(b),max(b),count(a) from z group by a where rownum<=10;
对a进行分组，计算平均b的平均、总和，最高，行数。
注意：查询字段中有sum、AVG、max类似的聚合函数，那么其他字段必须要在group by中出现，通俗的说group by后面的字段不能少于查询中的非聚合字段。

9.子查询
在执行数据操作中，如果某个操作需要依赖于另一个select语句的查询结果，那么就可以吧select语句嵌入到该操作语句中，这样就形成了一个子查询，子查询通常用于where子句中。
在使用子查询时，应注意以下规则：
a.子查询必须用“()”括起来
b.子查询中不能包含order by子句（order by子句只能对最终结果排序）
c.子查询允许嵌套多层，但不能超过255层
